Error on "BullseyeCoverageTemplateSyntaxError"

SYMPTOMS

A compiler error containing the symbol BullseyeCoverageTemplateSyntaxError occurs, such as one of the following.

Borland C++Error E2451 ...: Undefined symbol 'BullseyeCoverageTemplateSyntaxError'
CodeWarriorundefined identifier 'BullseyeCoverageTemplateSyntaxError'
IBM C++1540-0274 (S) The name lookup for "BullseyeCoverageTemplateSyntaxError" did not find a declaration.
Intel C++error: identifier "BullseyeCoverageTemplateSyntaxError" is undefined
Microsoft C++error C2065: 'BullseyeCoverageTemplateSyntaxError' : undeclared identifier
SGI MIPSpro C++cc-1020 CC: ERROR ... The identifier "BullseyeCoverageTemplateSyntaxError" is undefined.
Sun C++Error: BullseyeCoverageTemplateSyntaxError is not defined.

CAUSE

BullseyeCoverage encountered a syntax error in the template function where the error occurred.

RESOLUTION

Compile your code without BullseyeCoverage. If a syntax error occurs, fix the source code. If no syntax error occurs, report this problem to Technical Support.

MORE INFORMATION

The compilers listed below defer syntax checking of template functions until instantiation. However, BullseyeCoverage cannot determine whether template functions are instantiated. In order to follow the behaviour of these compilers, BullseyeCoverage handles a syntax error in a template function by inserting the undefined identifier BullseyeCoverageTemplateSyntaxError into the function. If the function is instantiated, an error is guaranteed to occur, ensuring that BullseyeCoverage parses code the same way as the compiler.

These compilers defer syntax checking of template functions until instantiation.

  • Borland C++
  • CodeGear C++
  • CodeWarrior
  • GCC 3.3 and earlier
  • HP aC++ 6
  • IBM C++
  • Intel C++
  • Microsoft C++
  • SGI MIPSpro
  • Sun C++

These compilers do not defer syntax checking.

  • Embarcadero C++ 6.20 and later
  • GCC 3.4 and later
  • HP aC++ 3

You can demonstration the compiler behavior with the source code below. The source contains an error that is not reported. Uncomment the first line in main to see the error reported. 

template<typename T>
void f()
{
    error
}

int main()
{
    //f<int>();
    return 0;
}
