Visual C++ Does Not Rebuild After Modifying Header

SYMPTOMS

The Microsoft Visual C++ Build command does not re-compile any sources even though a header file in the project was modified.

CAUSE

Microsoft Visual C++ fails to record dependency information when preprocessing is performed separately from compiling, as BullseyeCoverage does.

RESOLUTION

After modifying a header, rebuild your whole project.

MORE INFORMATION

The Microsoft Visual C++ compiler generates dependency information into a file named project.pdb during preprocessing. BullseyeCoverage feeds the compiler preprocessed source files. Microsoft Visual C++ records only dependency information for the main source file but not the header files named by #line directives.

Microsoft does not document the .pdb file format, so BullseyeCoverage cannot work around the problem by updating the file with the correct dependencies.

The problem does not occur with Microsoft Visual C++ version 6.
