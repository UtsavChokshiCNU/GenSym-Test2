Error LNK1104: cannot open file libcov-windows-cpu.lib

SYMPTOMS

fatal error LNK1104: cannot open file "libcov-windows-cpu.lib"

CAUSE

Coverage Build was enabled when you built one or more of your object files or library files, but BullseyeCoverage is not installed during linking.

RESOLUTION

Delete all object files and library files which might have been built with Coverage Build enabled. You can find these files using Start|Find|Files and Folders specifying Containing text as libcov-windows-cpu.lib. and Look in as your root directory. Search all your disks and network drives.

Alternatively, you can determine which object files were built with Coverage Build enabled by temporarily adding link option -nodefaultlib:libcov-windows-cpu.lib. This results in LNK2001 or LNK2019 errors which contain the name of the problematic object file(s). In the example below, calc1.obj was built with BullseyeCoverage. 

calc1.obj: error LNK2001: unresolved external symbol _cov_probe_x_y_z
