Freescale CodeWarrior

The CodeWarrior integrated development environment cannot be configured to build using BullseyeCoverage. You can build with BullseyeCoverage using the command line compiler.

Freescale CodeWarrior for ColdFire

This information applies to Freescale CodeWarrior for CodeWarrior version 7.1.

Build Run-Time Library

Use the commands below to build the BullseyeCoverage run-time library. Modify the -arch and -be options to match your project. 

set CWFolder=%ProgramFiles%/Freescale/CodeWarrior for ColdFire V7.1
set PATH=%CWFolder%/ColdFire_Tools/Command_Line_Tools;%PATH%
set MWCIncludes=%CWFolder%/ColdFire_Support/msl/MSL_C/MSL_Common/Include;%CWFolder%/ColdFire_Support/msl/MSL_C/MSL_ColdFire/Include
cd %ProgramFiles%\BullseyeCoverage\run
mwccmcf -c -proc MCF5407 -O libcov-coldfire.c
mwldmcf -library -o ../lib/libcov-coldfire.a libcov-coldfire.o

Freescale CodeWarrior for Power Architecture

This information applies to Freescale CodeWarrior for Power Architecture version 8.8.

BullseyeCoverage supports this compiler. The Freescale run-time library does not provide the infrastructure required by the BullseyeCoverage run-time. Therefore, you must port the BullseyeCoverage run-time to your target platform. See Custom and Embedded Environments.

Freescale CodeWarrior for StarCore

This information applies to Freescale CodeWarrior for StarCore version 3.2.1.

Build Run-Time Library

Use the commands below to build the BullseyeCoverage run-time library. Modify the -arch and -be options to match your project. 

cd %ProgramFiles%\BullseyeCoverage\run
scc -arch sc3400 -be -c -O libcov-starcore.c
sc100-ar -cr ../lib/libcov-starcore.elb libcov-starcore.eln
