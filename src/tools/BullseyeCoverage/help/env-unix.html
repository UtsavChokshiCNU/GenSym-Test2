Unix

Building the Run-Time Library

You do not need to build the run-time library for the host operating system; the library is supplied pre-compiled for most compilers.

If you are cross-compiling for a Unix/POSIX operating system, you must build the run-time library for your target. If you are using GCC, see GCC Cross Compiler, otherwise continue below.

To build the run-time library, move to the directory BullseyeCoverage/run. Modify the Makefile to use your cross compiler, and rebuild. For example: 

cd /opt/BullseyeCoverage/run
make

Environment Variables

If the BullseyeCoverage run-time cannot find an environment variable in the process environment, it first searches the file /tmp/BullseyeCoverageEnv.txt then the file /etc/BullseyeCoverageEnv.txt. These ASCII text files have the format below: 

name1=value1
name2=value2
...

This feature is intended for measuring coverage of programs launched during operating system boot, where setting environment variables is not practical.
