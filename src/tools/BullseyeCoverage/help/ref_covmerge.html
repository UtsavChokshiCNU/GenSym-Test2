covmerge - Merge Measurements 

covmerge [options] input.cov ...

DESCRIPTION

covmerge combines measurements in each of the coverage files input.cov into a result coverage file. You can use this command to combine measurements obtained when testing a program on separate computers. It is not recommended to merge files that resulted from compiling different source file versions. When covmerge encounters more than one version of the same source file (by file time stamp), covmerge discards measurements for the older versions.

You can specify multiple coverage files separated by commas. This allows specifying the value of the COVFILELIST environment variable directly on the command line.

OPTIONS

-c--create Create a new result coverage file rather than merging with the existing result coverage file.
-fPath--file Path The coverage file is specified by Path. The default is test.cov.
-h--help Display a summary of options.
-q--quiet Suppress warnings and the copyright message.
-v--verbose Show informational messages. This option displays the status of merging each input file.
--no-banner Suppress startup banner

EXAMPLES

The command below combines measurements obtained by two different testers into a new a coverage file. 

covmerge -c -ftotal.cov cathy.cov paul.cov

The commands below show how you can merge all the files specified by COVFILELIST, in Korn shell syntax. 

export COVFILELIST=file1.cov,file2.cov,file3.cov
covmerge -c -ftotal $COVFILELIST

ENVIRONMENT VARIABLES

COVFILEThe coverage file. Option -f (--file) overrides this setting.
COVMERGECFGConfiguration file. See FILES.

FILES

If the files listed below exist, they contain initial options separated by white space.

BullseyeCoverage/bin/covmerge.cfg
home-directory/covmerge.cfg

NOTES

If covmerge encounters measurements from the same source file with two different modification times, covmerge discards the older measurements.
