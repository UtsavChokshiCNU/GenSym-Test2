Coverage Build error: no compiler interceptors

SYMPTOMS

Coverage Build error: no compiler interceptors in BullseyeCoverage/bin

Coverage Build unavailable; no compiler interceptors in BullseyeCoverage/bin

CAUSE

Installation of BullseyeCoverage was not completed correctly.

RESOLUTION

You can either create compiler interceptors manually, or you can re-install BullseyeCoverage.

Creating Compiler Interceptors Manually on Windows

Move to the BullseyeCoverage/bin directory and copy the program covc.exe to the name of your compiler. For example, 

cd %ProgramFiles%\BullseyeCoverage\bin
copy covc.exe cl.exe

Creating Compiler Interceptors Manually on Unix

Move to the BullseyeCoverage/bin directory and link the program covc to the name of your compiler. For example, 

cd /opt/BullseyeCoverage/bin
ln covc cc

Re-installing on Windows

Re-run the installation program. At the page Choose Compilers, select all the compilers you are using.

Re-installing on Unix

  1. Make sure all your compilers are listed in the PATH environment variable.
  2. echo $PATH >install_path
  3. Become super user (if required to write to the target directory)
  4. ./install --prefix=directory
    You can install into the BullseyeCoverage distribution directory itself. To do this, use "--prefix=.".
