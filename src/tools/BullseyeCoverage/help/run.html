Running Test Programs

Running on the Build Computer

There are no special steps required to run a test program built with BullseyeCoverage on the local computer; run your program as normal. Be sure the COVFILE environment variable names your coverage file.

Running on Multiple Computers

There are two ways to collect and merge coverage data on multiple computers.

  1. Share one coverage file on a network by setting environment variable COVFILE on the remote computer to the network path of the coverage file. This alternative is simpler but requires adequate network performance.
  2. Copy the coverage file to each remote computer and set COVFILE on the remote computer to the path of the copy. After testing is complete, merge the coverage files into one file for reporting. Before making code changes or rebuilding, merge all copies of the coverage files back into the one on the build system. To merge coverage files using the Coverage Browser, use command File Insert. To merge coverage files at the command prompt, use covmerge.

Collecting Data to Multiple Coverage Files

If you have multiple, separate projects which run together on the same computer and each project has its own associated coverage file, you can collect coverage data for all the projects simultaneously without merging the coverage files. Set environment variable COVFILELIST to a comma separated list of the coverage file paths. For example, 

COVFILELIST=/path1/test1.cov,/path2/test2.cov,/path3/test3.cov

Environment variable COVFILELIST overrides COVFILE, and is only used at run-time.

Controlling Run-Time Error Display

If an error occurs at run-time, BullseyeCoverage displays an error message to the screen. However, if environment variable COVERR is set to the name of a file, BullseyeCoverage appends the message to the file instead.
