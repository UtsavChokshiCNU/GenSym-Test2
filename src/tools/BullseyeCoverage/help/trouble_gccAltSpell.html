Error: expected ']' with GCC -fpermissive

SYMPTOMS

With BullseyeCoverage enabled, the error below occurs. 

error: expected ']'

Without BullseyeCoverage, GCC displays the warning or note below. 

warning: `<::' cannot begin a template-argument list
note: `<:' is an alternate spelling for `['. Insert whitespace between `<' and `::'

CAUSE

A source file inadvertently contains <:, which is an alternate spelling for [. The GCC compile options include -fpermissive, which causes the compiler to interpret <:: as the two separate tokens < ::. For example: 

Template<::argument>

BullseyeCoverage interprets this as: 

Template[:argument>

RESOLUTION

Insert a space between the < and the :: tokens. For example: 

Template< ::argument>

Do not use -fpermissive.
