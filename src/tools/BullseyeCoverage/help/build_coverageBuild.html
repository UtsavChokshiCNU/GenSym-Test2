Coverage Build

Enabling and Disabling

To measure coverage, enable Coverage Build, then rebuild the source code you want to measure. You can enable and disable Coverage Build by any of the methods below.

  • Coverage Browser menu command Tools Coverage Build.
  • Microsoft Visual Studio menu command Tools Enable/Disable BullseyeCoverage Build.
  • The command line program cov01. To enable, use cov01 -1. To disable, use cov01 -0.

When Coverage Build is enabled, BullseyeCoverage instruments any project you build. You need not use the same method to enable as you use to disable Coverage Build. The Coverage Build setting is per-user; when one user changes the Coverage Build setting it does not affect the setting for other users. This is true regardless of whether you build from the command prompt or from within an integrated development environment. Enabling or disabling Coverage Build does not remove any object files or intermediate files.

Compiler Interceptors

The BullseyeCoverage bin directory contains compiler interceptors, which are programs named the same as a compiler program. When you build, control passes to the compiler interceptor instead of the real compiler. With Coverage Build enabled, the compiler interceptor invokes covc, which adds probes and then invokes the real compiler on the instrumented source. With Coverage Build disabled, the compiler interceptor simply passes control directly to the real compiler. You know Coverage Build is enabled when you see the startup banner below.
