Building with Borland C++ 5.02 and prior

The Borland C++ integrated development environment cannot be configured to build using BullseyeCoverage due to limitations of Borland C++. However, you can build from the command prompt using a makefile as described below.

In the integrated development environment, create a makefile from your project using the command Project Generate makefile. To ensure the generated makefile is correct, go to the command prompt and build the project without BullseyeCoverage as shown below. 

cov01 -0
erase *.obj
make -f project.mak

The integrated development environment often creates an invalid makefile. For assistance correcting the makefile, contact Borland Technical Support. After you have the makefile functioning correctly, rebuild the project with BullseyeCoverage as shown below. 

cov01 -1
erase *.obj
make -f project.mak
