GCC Cross Compiler

This information applies to cross compiling with GCC targeting user mode Linux/Unix or a custom environment.

Build Run-Time Library

Use commands like those shown below to build the BullseyeCoverage run-time library. Modify the compile command to use the same compiler options as your project. These commands are for a Bourne-compatible shell such as bash. 

cd BullseyeCoverage/run
cc=arm-linux-gcc    # customize to your tool chain
ar=arm-linux-ar     # customize to your tool chain

src=libcov-posix.c  # for Linux/Unix
#src=libcov-user.c  # for custom environment

$cc -c -D_REENTRANT -fPIC -O2 -o libcov-cross.o $src
$ar -rc ../lib/libcov-`$cc -dumpmachine`.a libcov-cross.o

Build Project

Make sure the BullseyeCoverage/bin is first in the PATH environment variable. Enable Coverage Build and build your project object files. For example: 

export PATH=/opt/BullseyeCoverage/bin:$PATH
cov01 -1
make

Running on Linux/Unix

Before running your test program, set environment variable COVFILE on the target device to the path of your coverage file located on your host system. You must have read and write access to the coverage file. For example: 

$ export COVFILE=/host/project/test.cov
$ ./a.out
